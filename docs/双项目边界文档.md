# Open Claw / RainbowPaw 双项目边界文档

**版本**: V1.0

**目标**: 明确两套代码、两个 Bot、两类业务的职责边界，避免功能重复、逻辑串线、维护失控。

## 1. 项目概述

### 项目A: rainbowpay_claw_Bot
**定位：**
- 低门槛互动入口
- 用户首次付费入口
- 抽奖 / 拼团 / 分销 / 钱包主阵地

**目标：**
- 拉新
- 激活
- 首次成交
- 裂变

### 项目B: RainbowPawbot
**定位：**
- 商城承接
- 纪念与善终服务承接
- 高客单和长期价值服务入口

**目标：**
- 商城转化
- 纪念服务
- 善终服务
- 墓位 / 年费服务
- 客服接管

## 2. 项目边界总原则
- **原则1：前台业务分离**
  两个 Bot 面向不同用户心智，前台交互不强行统一。
- **原则2：后台能力统一**
  钱包、积分、用户身份、bridge、报表、风控统一。
- **原则3：Claw Bot 负责“入口与交易”**
- **原则4：RainbowPawbot 负责“承接与高价值服务”**

## 3. 功能归属表

| 功能 | rainbowpay_claw_Bot | RainbowPawbot | 说明 |
| --- | --- | --- | --- |
| 用户注册 | 主 | 可读 | 统一走 global_user_id |
| 抽奖 | 主 | 否 | 只在 claw-bot |
| 支付凭证提交 | 主 | 否 | 抽奖支付在 claw-bot |
| 钱包展示 | 主 | 可展示 | 钱包服务统一 |
| 积分发放 | 主 | 可消费 | 统一 wallet-service |
| 拼团 | 主 | 否 | 主逻辑只在 claw-bot |
| 分销 | 主 | 否 | 主逻辑只在 claw-bot |
| 商城浏览 | 可导流 | 主 | claw-bot 只导流 |
| 商品购买 | 可简化 | 主 | 正式商城以 RainbowPawbot 为主 |
| 纪念服务 | 导流 | 主 | claw-bot 不承接细节 |
| 善终服务 | 导流 | 主 | RainbowPawbot 承接 |
| 墓位/年费 | 否 | 主 | 高客单服务 |
| 客服接管 | 基础 | 主 | 复杂咨询在 RainbowPawbot |
| 用户画像标签 | 写入 | 写入/读取 | 统一 bridge-service |
| 报表 | 读取 | 读取 | 统一 report-service |
| 风控 | 触发 | 触发 | 统一 risk-service |

## 4. Claw Bot 明确负责内容

**必须负责：**
- 抽奖
- 抽奖奖池
- 抽奖次数控制
- 支付凭证审核
- 拼团
- 分销
- 钱包/积分入口
- 基础订单（抽奖商品）
- 首次转化
- 基础用户标签

**禁止负责：**
- 善终完整服务咨询
- 墓位完整购买流程
- 复杂商城页面
- 长期纪念服务主流程
- 高价值售后承接

## 5. RainbowPawbot 明确负责内容

**必须负责：**
- 商城主承接
- 纪念服务
- 善终服务
- 墓位/年费服务
- 纪念页
- 高价值客户承接
- 人工客服升级
- 高客单商品与服务

**禁止负责：**
- 抽奖核心逻辑
- 拼团主逻辑
- 分销主逻辑
- 抽奖奖池管理
- 抽奖支付审核

## 6. 两项目交互规则

**场景1：Claw Bot → RainbowPawbot**
- **触发条件：**
  - 累计消费达到阈值
  - 用户点击纪念/商城/善终入口
  - 用户标签命中 memorial_interest / high_value_user
- **方式：**
  - 通过 bridge-service 生成 deep link
  - 带 scene 参数跳转

**场景2：RainbowPawbot ← Claw Bot 用户画像**
- **读取内容：**
  - pet_type
  - spend_total
  - spend_level
  - interest_tags
  - last_active_at

**场景3：商城消费返还**
- 商城消费产生的积分、余额变化统一写入 wallet-service，两个 Bot 都能读取。

## 7. 禁止事项
- **禁止1**：两个项目分别维护独立的钱包体系
- **禁止2**：两个项目分别维护不同的积分换算规则
- **禁止3**：两个项目直接互调数据库内部表，不经过公共服务
- **禁止4**：Claw Bot 私自承接善终服务完整流程
- **禁止5**：RainbowPawbot 私自复制抽奖/拼团/分销逻辑

## 8. 决策规则：新功能判断流程
新需求进来后，先问：
1. **这个需求是“低门槛引流/互动/首次付费”吗？**
   - 是 → 归 Claw Bot
2. **这个需求是“商城/纪念/善终/长期服务”吗？**
   - 是 → 归 RainbowPawbot
3. **这个需求是否涉及用户资产、用户身份、风控、报表？**
   - 是 → 归公共服务

## 9. 发布影响检查
每次改动前必须回答：
- 改的是哪个项目？
- 是否影响 wallet-service？
- 是否影响 bridge-service？
- 是否影响管理员报表？
- 是否影响另一个 Bot 的用户体验？
