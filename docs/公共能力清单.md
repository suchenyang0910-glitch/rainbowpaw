# Open Claw / RainbowPaw 公共能力清单

**目标**: 梳理哪些能力必须统一，哪些能力暂时分开，方便开发和运维。

## 1. 公共能力分类
A. 已统一
B. 必须统一（待做）
C. 可选统一（后做）
D. 保持分离

## 2. 已统一（示例）
| 能力 | 状态 | 说明 |
| --- | --- | --- |
| global_user_id | 已统一 | 两个 Bot 用户统一身份 |
| bridge deep link | 已统一 | 通过 token 跳转 |
| 报表命令 | 已统一 | `/report`、`/profit` 等统一输出 |
| 报警系统 | 已统一 | Bot 发管理员 |

## 3. 必须统一（待做）

### 3.1 用户身份中心
- **服务名**: identity-service
- **功能**:
  - global_user_id
  - telegram_id 映射
  - bot_user_mapping
- **状态**: 待做 / 开发中 / 已完成
- **优先级**: P0

### 3.2 钱包/积分中心
- **服务名**: wallet-service
- **功能**:
  - locked points
  - cashable points
  - credit balance
  - 提现申请
  - 流水日志
- **优先级**: P0

### 3.3 bridge-service
- **功能**:
  - deep link token
  - 用户画像查询
  - 标签同步
  - 事件写入
- **优先级**: P0

### 3.4 report-service
- **功能**:
  - 收入
  - 利润率
  - 抽奖数据
  - 拼团数据
  - bridge 导流数据
  - 每日报表
  - 报警推送
- **优先级**: P0

### 3.5 risk-service
- **功能**:
  - 异常中奖
  - 异常提现
  - 异常拼团
  - 同设备 / 同IP 风控
  - 黑名单 / 冻结
- **优先级**: P1

## 4. 可选统一（后做）

### 4.1 AI 调用封装
- **功能**:
  - 统一 Prompt 模板
  - 统一模型调用
  - 统一日志
- **优先级**: P2

### 4.2 推送消息中心
- **功能**:
  - 拼团提醒
  - 积分过期提醒
  - 召回消息
  - 活动消息
- **优先级**: P2

### 4.3 商品推荐引擎
- **功能**:
  - 抽奖后推荐商品
  - 根据标签推荐商城入口
  - 导流 RainbowPawbot
- **优先级**: P2

## 5. 保持分离的能力
这些不要强行统一：

| 能力 | 原因 |
| --- | --- |
| Bot 菜单结构 | 两边心智不同 |
| 抽奖交互流程 | 只属于 Claw Bot |
| 善终咨询流程 | 只属于 RainbowPawbot |
| 商城承接文案 | 服务语境不同 |
| 客服脚本 | 一个偏游戏，一个偏服务 |
