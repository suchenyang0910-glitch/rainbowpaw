# P2 增强层 PRD（10% 资源）

> 目标：在不干扰 P0 现金流主线的前提下，提升互动与 ARPU。

## P2 原则（上线边界）
- 不允许引入“不可控复杂度”（新状态机/新支付通道/重构全链路）。
- P2 的任何改动必须：
  - 不影响 P0 的建档/推荐/下单/通知/预约
  - 有清晰的开关与回滚路径

## P2 功能清单

## A. Claw 抽奖（作为流量外挂）
### A1 最小健康度
- 抽奖可玩、可回收
- 幂等与扣费准确
- 中奖后能跳转到商城（导流变现）

验收：
- 100 次抽奖无重复扣费；回收可用；中奖后导流按钮可用。

## B. 钱包积分高级玩法（轻量）
### B1 资产结构清晰
- 只保留 2 类：`locked`（不可提现）与 `cashable`（可提现）
- 明确扣减策略：优先扣 locked

### B2 手动对账入口
- 管理员可查看某用户积分变动明细（用于客服处理）

验收：
- 客服能回答“为什么我积分变了/可以提现吗”。

## C. 自动优惠券（最小）
### C1 发券触发
- 新用户建档完成
- 邀请完成
- 放弃支付 24h

### C2 核销规则
- 仅作用于指定 SKU/套餐

验收：
- 发券与核销有日志，且可回溯。

## D. 会员等级（最小）
### D1 规则
- 按累计消费/订阅时长分 2-3 级

### D2 权益
- 仅做“折扣/优先客服/生日礼”等轻权益

验收：
- 等级变更可追踪；不会影响订单/支付。
