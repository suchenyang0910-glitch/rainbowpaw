# RainbowPaw 投资人简报（Investor Brief）

## 1. 一句话（3 套口径）
### 用户版
RainbowPaw：宠物护理、善终与纪念一站式平台。

### 投资人版
RainbowPaw uses gamified acquisition + pet data + recurring commerce to monetize high-margin aftercare services.

### 团队内部版
低成本获客，高复购变现，高毛利收口。

## 2. 我们解决什么问题
### 用户侧
- 决策难：宠物健康/护理用品选择信息不对称，用户难以判断“买什么、怎么用、怎么买更划算”。
- 情绪与服务缺口：宠物离世后的善终与纪念需求强，但服务供给零散、体验不一致、信任成本高。

### 供给侧（商家/服务方）
- 线上获客贵：单点电商转化低、复购弱，CAC 高。
- 服务交付难：善终/纪念服务需要强运营与流程化，缺少统一的订单/支付/客服/履约工具。

## 3. 解决方案（产品形态）
RainbowPaw 不是单一电商或抽奖产品，而是一个“能赚钱的作战系统”：

一句话记忆点：我们不是卖商品，我们是在经营宠物家庭关系周期。

### 3.1 双 Bot + 双 MiniApp
- Claw Bot + Claw MiniApp（低门槛互动入口）：抽奖玩法、邀请裂变、抽奖次数购买。
- RainbowPaw Bot + RainbowPaw MiniApp（高价值承接）：商城/纪念馆/善终服务入口、订单与履约。
- Bridge 深链路：在两个 Bot/小程序间可控跳转，确保用户从低门槛场景自然进入高客单服务。

### 3.2 数据与运营中台
- 宠物画像：品类（猫/狗）、年龄、体重、疾病等作为推荐与分层依据。
- 钱包/积分体系：抽奖次数、奖励回收、积分锁定/可提现吗等经济模型，驱动留存与复购。
- CRM 与自动化：报价、跟进、提醒、转化漏斗，支撑善终服务这种“高信任成交”。

## 4. 商业模式（更“钱味”的结构）
### 收入结构（A-E）
- A. 流量收入：抽奖次数（Plays）购买，小额高频，驱动互动与裂变。
- B. 现金流收入：保健品/护理包订阅（示例：$29-$50/月），稳定复购现金流。
- C. 毛利收入：善终/纪念服务（高客单高毛利），由 CRM 与 SOP 承接转化。
- D. 长尾收入：纪念体系（纪念馆、纪念品、周年提醒等）带来长期留存与复购。
- E. 数据收入（未来 B 端）：基于画像与生命周期数据的供给侧合作、精准投放与服务分发。

## 6. 护城河（资本语言版）
1) 获客护城河：传统宠物电商买流量，我们通过抽奖互动 + 内容自己制造流量。
2) 数据护城河：宠物年龄、疾病、消费能力、生命周期与支付行为数据是稀缺资产。
3) 信任护城河：用户在最脆弱阶段选择你，一旦成交，忠诚度和复购率显著更高。
4) 服务护城河：善终服务本地履约复杂、复制慢，SOP 与供给网络构成壁垒。

## 8. 技术与系统（保证可规模化）
### 架构概览
- 微服务 + API Gateway：统一 `/api` 对外，内部拆分 identity/wallet/order/bridge/risk/report/ai 等服务。
- PostgreSQL 分 schema：按域隔离（identity、wallet、order、claw、bridge、store、service、payments 等）。
- Web：React + Vite（官网、多语言、Admin Console、MiniApp）。
- Bots：Node.js Telegram bots（claw/rainbow/admin）。
- 部署：Docker Compose，自托管；Caddy/Nginx 反向代理与 HTTPS。

### 支付与对账（示例：USDT）
- SettleCore：创建支付单 → 用户转账 → webhook 回调 → 入库对账 → 钱包/订单状态更新。
- 管理通知：用户提交支付凭证后可推送管理员 Telegram，支持人工审核/客服介入。

## 11. 投资人会问的 3 个问题（标准答案）
Q1 为什么用户要玩抽奖？
- 低门槛互动，提高首次进入率与数据采集效率。

Q2 为什么不直接做电商？
- 宠物电商获客成本高，我们先获得关系与画像，再卖商品与订阅，ROI 更好。

Q3 为什么能做大？
- 我们做的是宠物家庭生命周期平台，不是单次交易平台；善终服务的交付网络与信任资产天然形成壁垒。

## 12. 关键指标（建议对投资人披露的口径）
（以下为模板，可填入真实数据）
- Acquisition：DAU/MAU、CAC、渠道结构、点击→入群/进入 MiniApp 转化率。
- Engagement：抽奖参与率、分享率、留存（D1/D7/D30）。

## 13. 风险与应对
- 合规与平台策略：Telegram/支付通道政策变化 → 多通道与可替代方案。
- 供给侧交付风险：服务不稳定 → 标准化流程、合作方筛选与 SLA。
- 资金与汇兑：跨境收款 → 账务对账、风控与资金安全策略。
