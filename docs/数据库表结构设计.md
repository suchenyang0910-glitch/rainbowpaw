# 数据库表结构设计 (PostgreSQL)

建议采用一个 PostgreSQL 实例，划分三个 schema：`identity`, `wallet`, `bridge`。

## 1. identity schema

### 1.1 global_users (统一身份主表)
```sql
CREATE SCHEMA IF NOT EXISTS identity;

CREATE TABLE identity.global_users (
  id BIGSERIAL PRIMARY KEY,
  global_user_id VARCHAR(64) NOT NULL UNIQUE,
  telegram_id BIGINT,
  username VARCHAR(128),
  first_source VARCHAR(64),
  primary_bot VARCHAR(32),
  pet_type VARCHAR(16),
  spend_total NUMERIC(12,2) NOT NULL DEFAULT 0,
  spend_level VARCHAR(16) NOT NULL DEFAULT 'low',
  last_active_at TIMESTAMP,
  status VARCHAR(16) NOT NULL DEFAULT 'active',
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_global_users_telegram_id ON identity.global_users(telegram_id);
CREATE INDEX idx_global_users_global_user_id ON identity.global_users(global_user_id);
```
**字段说明:**
- `global_user_id`: 平台统一用户 ID，建议格式 `g_xxxxx`
- `first_source`: `tiktok` / `telegram_group` / `referral` / `claw_bot`
- `primary_bot`: `claw` / `rainbowpaw`
- `pet_type`: `cat` / `dog` / `unknown`
- `spend_level`: `low` / `mid` / `high`
- `status`: `active` / `frozen` / `banned`

### 1.2 bot_user_mapping (Bot 用户映射表)
两个 Bot 自己的 user_id 映射到统一 ID。
```sql
CREATE TABLE identity.bot_user_mapping (
  id BIGSERIAL PRIMARY KEY,
  global_user_id VARCHAR(64) NOT NULL,
  source_bot VARCHAR(32) NOT NULL,
  source_user_id VARCHAR(64) NOT NULL,
  telegram_id BIGINT,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE (source_bot, source_user_id)
);

CREATE INDEX idx_bot_user_mapping_global_user_id ON identity.bot_user_mapping(global_user_id);
CREATE INDEX idx_bot_user_mapping_telegram_id ON identity.bot_user_mapping(telegram_id);
```
**`source_bot` 建议值:**
- `claw_bot`
- `rainbowpaw_bot`

### 1.3 user_tags (统一标签表)
```sql
CREATE TABLE identity.user_tags (
  id BIGSERIAL PRIMARY KEY,
  global_user_id VARCHAR(64) NOT NULL,
  tag_key VARCHAR(64) NOT NULL,
  tag_value VARCHAR(128),
  score NUMERIC(8,2) NOT NULL DEFAULT 1,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE (global_user_id, tag_key)
);

CREATE INDEX idx_user_tags_global_user_id ON identity.user_tags(global_user_id);
```
**`tag_key` 示例:**
- `pet_type`
- `toy_interest`
- `store_interest`
- `memorial_interest`
- `aftercare_interest`
- `high_value_user`

## 2. wallet schema

### 2.1 wallets (统一钱包主表)
```sql
CREATE SCHEMA IF NOT EXISTS wallet;

CREATE TABLE wallet.wallets (
  id BIGSERIAL PRIMARY KEY,
  global_user_id VARCHAR(64) NOT NULL UNIQUE,
  points_total NUMERIC(12,2) NOT NULL DEFAULT 0, -- 总积分
  points_locked NUMERIC(12,2) NOT NULL DEFAULT 0, -- 不可提现积分，可消费
  points_cashable NUMERIC(12,2) NOT NULL DEFAULT 0, -- 可提现积分
  wallet_cash NUMERIC(12,2) NOT NULL DEFAULT 0, -- 提现余额，美元
  total_earned NUMERIC(12,2) NOT NULL DEFAULT 0,
  total_spent NUMERIC(12,2) NOT NULL DEFAULT 0,
  status VARCHAR(16) NOT NULL DEFAULT 'active',
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_wallets_global_user_id ON wallet.wallets(global_user_id);
```
**字段说明:**
- `points_total`: 总积分 (locked + cashable)
- `points_locked`: 不可提现积分，可消费
- `points_cashable`: 可提现积分
- `wallet_cash`: 提现余额，美元 (用于提现通道)

### 2.2 wallet_logs (钱包流水表)
```sql
CREATE TABLE wallet.wallet_logs (
  id BIGSERIAL PRIMARY KEY,
  global_user_id VARCHAR(64) NOT NULL,
  biz_type VARCHAR(32) NOT NULL,
  asset_type VARCHAR(32) NOT NULL,
  change_direction VARCHAR(16) NOT NULL,
  amount NUMERIC(12,2) NOT NULL,
  balance_before NUMERIC(12,2) NOT NULL,
  balance_after NUMERIC(12,2) NOT NULL,
  ref_type VARCHAR(32),
  ref_id VARCHAR(64),
  remark TEXT,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_wallet_logs_global_user_id ON wallet.wallet_logs(global_user_id);
CREATE INDEX idx_wallet_logs_ref_type_ref_id ON wallet.wallet_logs(ref_type, ref_id);
```
**枚举示例:**
- `biz_type`: `recharge`, `claw_consume`, `claw_recycle`, `group_reward`, `referral_reward`, `store_consume`, `withdraw_apply`, `withdraw_success`, `withdraw_reject`
- `asset_type`: `points_locked`, `points_cashable`, `wallet_cash`
- `change_direction`: `in`, `out`

### 2.3 withdraw_requests (提现申请表)
```sql
CREATE TABLE wallet.withdraw_requests (
  id BIGSERIAL PRIMARY KEY,
  global_user_id VARCHAR(64) NOT NULL,
  request_no VARCHAR(64) NOT NULL UNIQUE,
  points_cashable_amount NUMERIC(12,2) NOT NULL,
  cash_amount NUMERIC(12,2) NOT NULL,
  fee_amount NUMERIC(12,2) NOT NULL DEFAULT 0,
  actual_cash_amount NUMERIC(12,2) NOT NULL,
  method VARCHAR(32) NOT NULL,
  account_info JSONB,
  status VARCHAR(16) NOT NULL DEFAULT 'pending',
  reviewed_by VARCHAR(64),
  reviewed_at TIMESTAMP,
  remark TEXT,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_withdraw_requests_global_user_id ON wallet.withdraw_requests(global_user_id);
CREATE INDEX idx_withdraw_requests_status ON wallet.withdraw_requests(status);
```
**`status` 示例:** `pending`, `approved`, `rejected`, `paid`

## 3. bridge schema

### 3.1 bridge_events (桥接事件流水表)
```sql
CREATE SCHEMA IF NOT EXISTS bridge;

CREATE TABLE bridge.bridge_events (
  id BIGSERIAL PRIMARY KEY,
  event_name VARCHAR(64) NOT NULL,
  global_user_id VARCHAR(64) NOT NULL,
  source_bot VARCHAR(32) NOT NULL,
  source_user_id VARCHAR(64),
  telegram_id BIGINT,
  event_data JSONB,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_bridge_events_global_user_id ON bridge.bridge_events(global_user_id);
CREATE INDEX idx_bridge_events_event_name ON bridge.bridge_events(event_name);
```
**`event_name` 示例:** `user_registered`, `pet_type_selected`, `first_payment_completed`, `play_completed`, `order_created`, `order_completed`, `interest_tag_updated`

### 3.2 deep_link_tokens (Deep Link Token 表)
```sql
CREATE TABLE bridge.deep_link_tokens (
  id BIGSERIAL PRIMARY KEY,
  token VARCHAR(128) NOT NULL UNIQUE,
  global_user_id VARCHAR(64) NOT NULL,
  from_bot VARCHAR(32) NOT NULL,
  to_bot VARCHAR(32) NOT NULL,
  scene VARCHAR(32) NOT NULL,
  extra_data JSONB,
  expires_at TIMESTAMP,
  used_at TIMESTAMP,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_deep_link_tokens_global_user_id ON bridge.deep_link_tokens(global_user_id);
CREATE INDEX idx_deep_link_tokens_token ON bridge.deep_link_tokens(token);
```
**`scene` 示例:** `store`, `memorial`, `aftercare`, `vip`, `cemetery`
