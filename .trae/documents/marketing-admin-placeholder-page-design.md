# 营销/风控占位模块后台页面设计（Desktop-first）

## Global Styles
- Layout：经典后台布局“顶部栏 + 左侧导航 + 右侧内容区”。内容区采用 12 列栅格 + Card 分组，表格为核心信息承载。
- Design tokens：
  - 背景：#F5F7FA（页面）；#FFFFFF（卡片）
  - 主色：#1677FF；危险：#FF4D4F；成功：#52C41A；警告：#FAAD14
  - 字体：基准 14px；标题 16/20；ID/subjectId 使用等宽字体
  - 交互：所有“启用/停用/发布/下线/撤销/移除”均需二次确认；所有写操作必须填写“变更原因”（Modal/Drawer 内必填）
- 权限态：
  - 模块 Tab：无权限不展示
  - 行为按钮：无权限隐藏或 disabled（统一策略）
  - 禁止态：命中黑名单校验（若开启）时，写按钮置灰并展示命中提示

## Meta Information（全局）
- title 模板：{页面名} - 运营后台
- description：用于配置拼团、分销、奖励、黑名单与活动配置的内部运营系统
- OpenGraph：og:title 同 title；og:type=website

## 页面：登录页（/login）
- Layout：居中卡片（宽 360~420px），上下留白 15vh。
- Page Structure：Logo/系统名 + 描述文案 + 登录按钮 + 错误提示区。
- Sections & Components：
  - 主按钮「使用统一账号登录」：跳转 SSO
  - 错误提示：回跳失败、权限不足、会话过期

## 页面：运营控制台（/ops）
- Layout：
  - 外层：Flex（左侧导航固定 240px，右侧自适应）
  - 内容区：顶部为页面标题与全局操作区；主体为“模块 Tab + 筛选区 + 表格区 + 详情/编辑抽屉”
- Page Structure：
  1) 顶部：页面标题「运营控制台」+ 右侧“审计入口”
  2) 第二层：模块 Tab（活动配置 / 拼团 / 分销 / 奖励 / 黑名单）
  3) 主体：
     - 筛选 Form（2 行内收敛，更多条件折叠）
     - Table（分页、排序、列显隐可选）
     - Drawer：详情/创建/编辑（同一抽屉复用）

### 通用组件规范
- 筛选区（Form）：
  - 左对齐，常用字段放第一行；搜索/重置固定在右侧
  - Enter 触发搜索；搜索后保留条件
- 表格（Table）：
  - 首列为名称/主体标识；末列为操作列
  - 操作列仅放 2~3 个高频动作，其余放「更多」下拉
- 抽屉（Drawer）：
  - 顶部显示状态 Tag（draft/active/inactive）与版本号（活动配置）
  - 表单分组：基础信息 / 规则 / 生效与风险提示
  - 底部固定按钮：取消、保存草稿（如适用）、发布/启用（主按钮）

### Tab：活动配置
- 列表字段：名称、类型、时间窗、范围摘要、状态、版本、更新时间
- 筛选字段：类型、状态、时间窗（起止）、关键词
- 行为：新建、编辑、发布、下线
- 关键交互：
  - 发布前弹窗预览：显示将生效的时间窗与范围摘要，并要求填写变更原因

### Tab：拼团
- 列表字段：活动名、关联配置、成团人数、有效期、库存、状态、更新时间
- 筛选字段：状态、关联配置、关键词
- 行为：新建、编辑、启用、停用
- 关键交互：
  - 启用时校验：关联配置必须为 active 且当前时间在时间窗内（否则给出阻断提示）

### Tab：分销
- 子 Tab（两列结构或二级 Tab）：
  1) 分销规则
  2) 分销员
- 分销规则：列表字段（名称、层级数、结算周期、状态、更新时间）；行为（新建/编辑/启用/停用）
- 分销员：列表字段（subjectType、subjectId、等级、状态、创建时间）；行为（新增/禁用/批量导入导出）

### Tab：奖励
- 子 Tab：
  1) 奖励策略
  2) 发放流水
- 奖励策略：列表字段（名称、类型、数值、状态、更新时间）；行为（新建/编辑/启用/停用）
- 发放流水：列表字段（主体、策略、金额、状态、时间、备注）；行为（手动发放、撤销）
- 手动发放弹窗/抽屉：选择策略 + 输入主体标识 + 金额（可覆盖或只读）+ 备注（必填）

### Tab：黑名单
- 列表字段：主体类型、主体标识、原因、有效期、状态、创建时间
- 筛选字段：主体类型、状态、关键词、是否即将过期（7 天）
- 行为：添加、移除、批量导入导出、立即校验
- 关键交互：
  - 添加/移除均要求原因；移除后状态变为 removed 并保留记录

## 页面：审计日志（/audit）
- Layout：顶部筛选 + 下方表格 + 详情抽屉。
- 筛选字段：时间范围、操作人、模块、动作、结果（成功/失败）、targetId。
- 表格字段：时间、操作人、模块、动作、targetId、结果。
- 抽屉内容：diffJson 以 JSON viewer 展示，支持复制；显示变更原因（从请求头或字段落库）。
